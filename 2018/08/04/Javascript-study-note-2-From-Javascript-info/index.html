Function

A function declaraion looks like

  • Values passed to a function as parameters are copied to its local variables
  • A function may access outer variable, but it works only from inside out. The code outside of the function doesn’t see its local variables
  • A function can return a value. If it doesn’t then it result is undefined

To make the code clean and easy to understand, it’s recommended to use mailnly local variables and parameters in the funtion, not outer variable

It is always easier to understand a funtion which gets parameters, works with them and returns a result than a function which gets no parameters, but modifier outer variables as a side-effort

Objects

Object are used to store keyed collections of various data and more complex entities.
An object can be created with figure brackets {} with an optional list of properties. A property is a “key:value” pair, where key is a string and value can be anything

Object property keys may be either of string type, or of symbol type.

Garbage collection

  • Garbage collection is performed automatically. We cannot force or prevent it
  • Objects are retained in memory while they are reachable
  • Being referenced is not the same as being reachable(from a root). A pack of interlinked objects can become unreachable as a whole

Object Methods

  • Functions that are stored in object properties are called “method”.
  • Methods allow objects to act like object.doSomething().
  • Methods can reference the object as this

The value of this is defined at run-time

  • When a function is declared, it may use this, but that this has no value until the function is called
  • That function can be copied between objects
  • When a function is called in the method syntac: object.method(), the value of this during the call is object

Arrow function are special: they have no this. when this is accessed inside an arrow function, it is taken from outside
